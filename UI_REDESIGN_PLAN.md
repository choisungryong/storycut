## 0. 디자인 비전 & 크리에이티브 디렉션

### 브랜드 에센스

StoryCut은 단순한 AI 도구가 아니라 **"브라우저 안의 영화 스튜디오"**다.
모든 UI 결정은 이 메타포에서 출발한다.

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   "Every pixel should whisper: Cinema."                 │
│                                                         │
│   사용자가 StoryCut에 접속하는 순간,                      │
│   영화관 조명이 꺼지고 스크린이 켜지는 그 순간의           │
│   기대감과 몰입감을 느껴야 한다.                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 디자인 무드보드 키워드

| 축 | 키워드 |
|---|--------|
| **톤** | Cinematic Dark, Theatrical, Premium |
| **감정** | 기대감(Anticipation), 몰입(Immersion), 경이(Wonder) |
| **질감** | 필름 그레인, 빛 줄기(Light Leak), 보케(Bokeh) |
| **모션** | 시네마틱 페이드, 파랄랙스, 리빌(Reveal) |
| **타이포** | 대담한 헤드라인 + 우아한 본문, 한영 혼합 하모니 |
| **참고** | Apple TV+ 랜딩, Runway ML, Pika Labs, Netflix 인터페이스 |

### 핵심 디자인 원칙

1. **Cinematic First**: 모든 전환(transition)은 영화적 컷(cut)처럼. 하드 전환 대신 dissolve, wipe, reveal
2. **Show, Don't Tell**: 기능 설명보다 결과물 시연. 텍스트 50% 줄이고 비주얼 200% 늘리기
3. **Progressive Disclosure**: 복잡성을 단계적으로 노출. 첫 화면은 숨 쉴 공간(breathing room)을 주고, 스크롤하면 깊이가 드러남
4. **Emotional Pacing**: 랜딩 페이지는 영화 트레일러처럼 **기-승-전-결** 리듬을 가짐

## 1. 현재 문제점 진단

### 치명적 문제
- **랜딩 페이지 부재**: 앱 접속 시 바로 폼이 보임 → 제품 가치 전달 불가
- **모바일 미대응**: `@media` 쿼리 없음, 심사위원이 모바일로 볼 가능성 높음
- **단일 HTML 1985줄**: index.html에 모든 섹션 + 인라인 스타일 → 유지보수 난이도 극상

### 디자인 품질 문제
- 기능 중심 UI → "개발자가 만든 느낌"
- CTA(Call-to-Action) 전략 부재
- 신뢰 요소 없음 (사용자 수, 후기, 데모 영상)
- 브랜딩 약함 (로고, 컬러 시스템, 타이포그래피)
- **감성적 후킹(Emotional Hook) 완전 부재** → 심사위원의 "와!" 반응을 끌어낼 수 없음

### VMAI 대비 약점
| 항목 | VMAI | StoryCut |
|------|------|----------|
| 랜딩 페이지 | 전문 마케팅 페이지 (히어로/기능/후기/FAQ) | 없음 |
| 반응형 | 완벽 대응 | 미대응 |
| 시각적 세련도 | 그라디언트, 글래스모피즘, 마이크로 애니메이션 | 기본 다크 테마 |
| CTA 전략 | "무료로 시작하기" + 데모 영상 | 바로 폼 노출 |
| 신뢰 요소 | 사용자 수, 후기 카드, 사례 영상 | 없음 |
| 브랜딩 | 일관된 디자인 시스템 | 로고+버전 텍스트만 |
| **감성 디자인** | 보통 (SaaS 표준) | **없음** |
| **결과물 쇼케이스** | 일부 | **없음** |

## 2. StoryCut 경쟁 우위 (VMAI에 없는 것)

이 기능들을 **시각적으로 강조**해야 함:

1. **뮤직비디오 생성** - VMAI에 없는 킬러 피처
2. **캐릭터 일관성** - Master Anchor + 7-step LOCK (업계 최고 수준)
3. **B-roll 스톡 영상** - Pexels 자동 배치 (비용 절감 + 품질 향상)
4. **7가지 아트 스타일** - anime, webtoon, hoyoverse, cinematic, realistic, illustration, abstract
5. **실시간 씬 편집** - 스토리 리뷰 → 이미지 프리뷰 → 개별 재생성
6. **AI 가사 타이밍 싱크** - 음원 분석 → 가사 자동 매칭
7. **I2V 영상 변환** - 이미지를 실제 영상으로 (Veo 3.1)

## 2-1. VMAI와의 차별화 전략

### 디자인 컨셉 차이
| | VMAI | StoryCut |
|---|------|----------|
| **포지셔닝** | "AI 비디오 자동화 서비스" (SaaS) | **"AI 영상 크리에이션 스튜디오"** (Creator Tool) |
| **비주얼 톤** | 클린 SaaS, 미니멀 | **시네마틱 스튜디오**, 필름스트립/타임라인 모티프 |
| **히어로** | 텍스트 + CTA + 데모 영상 | **풀스크린 시네마틱 인트로** + 듀얼 모드 분기 |
| **기능 소개** | 일반적 카드 그리드 | **스크롤 연동 파이프라인 시각화** (Scroll-triggered) |
| **갤러리** | 없음 | **몰입형 스타일 갤러리** (라이트박스 + 비포/애프터) |
| **차별 피처** | 텍스트→영상 단일 | MV 생성, 캐릭터 일관성, B-roll, 가사 싱크 |

### 핵심 차별화 요소
1. **시네마틱 풀스크린 히어로**: 접속 시 영화 트레일러급 비주얼. 배경에 실제 생성 결과물 영상이 자동 재생되며, 텍스트가 시네마틱 타이밍으로 페이드인
2. **듀얼 모드 인터랙티브 분기**: "AI 스토리 영상" / "뮤직비디오" 두 갈래를 시각적 카드 플립(3D)으로 전환. 호버 시 해당 모드의 데모 영상이 카드 배경에 재생
3. **스크롤 연동 파이프라인**: "주제 입력 → AI 스토리 → 이미지 → 영상" 과정이 스크롤에 따라 단계별 애니메이션. 각 단계에서 실제 데이터가 "타이핑"되는 효과
4. **필름스트립 UI 언어**: 타임라인, 프레임, 컷 등 영상 제작 메타포를 UI 전반에 활용
5. **라이브 1-씬 체험**: 랜딩에서 로그인 없이 간단한 데모 체험 (1씬 무료 생성)

### VMAI에서 벤치마크할 것 (좋은 점은 배우되 다르게)
- 글래스모피즘 네비게이션 → 채용하되 **필름 그레인 텍스처** 오버레이로 차별화
- 사용자 신뢰 요소 → **"AI가 만든 실제 영상" 갤러리** + 생성 통계로 대체
- 모바일 반응형 → 필수 적용 (심사위원 모바일 접속 대비)
- FAQ 아코디언 → 동일 패턴 채용

## 6. 기술 스택 결정

### 옵션 A: 바닐라 HTML/CSS/JS (권장)
- **장점**: 현재 구조와 일관성, 추가 빌드 불필요, 배포 간단
- **단점**: 컴포넌트 재사용 수동
- **적합**: 해커톤 일정(~3주) 고려 시 가장 현실적

### 옵션 B: Tailwind CSS만 추가
- **장점**: 유틸리티 클래스로 빠른 스타일링, 반응형 내장
- **단점**: CDN 사용 시 빌드 불필요하지만 클래스가 길어짐
- **적합**: 디자인 속도가 중요한 경우

### 옵션 C: React/Next.js 전환
- **장점**: 컴포넌트 기반, 생태계
- **단점**: 전면 재작성 필요 (3주 내 불가능)
- **부적합**: 해커톤 일정에 맞지 않음

**결론: 옵션 A (바닐라) + CSS Custom Properties 디자인 시스템**

Tailwind CDN은 선택적 보조 사용. 핵심은 `design-tokens.css`로 일관된 디자인 변수 체계를 구축하고, 바닐라 CSS로 컴포넌트를 작성하는 것. 이렇게 하면 빌드 도구 없이도 디자인 시스템의 이점을 얻을 수 있다.

// Intersection Observer 기반 스크롤 리빌
const revealObserver = new IntersectionObserver((entries) => {
  entries.forEach((entry, index) => {
    if (entry.isIntersecting) {
      // stagger 딜레이: 자식 요소별 순차 등장
      const delay = entry.target.dataset.delay || 0;
      setTimeout(() => {
        entry.target.classList.add('visible');
      }, delay);
      revealObserver.unobserve(entry.target);
    }
  });
}, { threshold: 0.15, rootMargin: '0px 0px -60px 0px' });

document.querySelectorAll('.reveal').forEach(el => {
  revealObserver.observe(el);
});

function animateCounter(element, target, duration = 2000) {
  let start = 0;
  const increment = target / (duration / 16);
  const timer = setInterval(() => {
    start += increment;
    if (start >= target) {
      element.textContent = target.toLocaleString() + '+';
      clearInterval(timer);
    } else {
      element.textContent = Math.floor(start).toLocaleString();
    }
  }, 16);
}

| **발표/데모** | 라이브 데모 시나리오 (주제 입력 → 실시간 생성) | 진행률 UI가 곧 데모의 주인공 |

### 심사위원 여정 (User Journey) 시나리오

```
1. 심사위원이 URL 접속 (모바일 또는 데스크톱)
   → 풀스크린 시네마틱 히어로 + 쇼릴 배경 영상
   → "와, 이거 뭐지?" (기대감 형성)

2. 스크롤 → 듀얼 모드 카드 + 호버 데모
   → "스토리 영상이랑 뮤직비디오 둘 다 되네?"

3. 파이프라인 시각화 (스크롤 연동)
   → "주제 입력하면 이렇게 자동으로 만들어지는구나"

4. 킬러 피처 벤토 그리드 + 캐릭터 일관성 비포/애프터
   → "캐릭터 일관성이 이 정도면 대단하네"

5. 스타일 갤러리 탭 전환
   → "7가지 스타일! 아트 품질이 꽤 좋은데?"

6. "무료로 시작하기" 클릭 → 앱 진입
   → 깔끔한 입력 폼, 스타일 카드 선택
   → "사용하기 쉽네"

7. 라이브 데모 중 진행률 UI
   → 씬이 하나씩 페이드인
   → "실시간으로 만들어지는 게 보이네!"

8. 결과 영상 재생
   → "와, 진짜 이걸 AI가 만들었어?"
```
