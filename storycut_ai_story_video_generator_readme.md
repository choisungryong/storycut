# STORYCUT

**STORYCUT**은 텍스트로 된 이야기를 입력하면, 자동으로 **유튜브 업로드가 가능한 완성형 영상(MP4)**을 생성해주는 AI 기반 스토리 영상 제작 에이전트입니다.

이 프로젝트의 목적은 **AI가 채널을 운영하는 것**이 아니라,
**여러 사용자가 STORYCUT 에이전트를 활용해 각자 자신의 이야기 영상을 만들고 유튜브에 업로드하도록 돕는 제작 도구**를 제공하는 것입니다.

---

## 1. 프로젝트 개요

- 프로젝트명: **STORYCUT**
- 성격: AI 기반 스토리 영상 생성 도구 (MVP)
- 대상 사용자:
  - 유튜브 크리에이터
  - 스토리텔링 콘텐츠 제작자
  - 얼굴 노출 없이 영상 콘텐츠를 만들고 싶은 개인
- 최종 산출물:
  - 유튜브 업로드가 가능한 MP4 영상 (16:9 또는 9:16)

---

## 2. STORYCUT이 해결하는 문제

기존 영상 제작의 문제점:
- 이야기 작성, 이미지/영상 제작, 음성 녹음, 편집을 모두 직접 해야 함
- 시간과 비용이 많이 듦
- 초보 크리에이터에게 진입 장벽이 높음

STORYCUT의 해결 방식:
- 사용자는 **아이디어와 옵션만 선택**
- 제작 과정은 **AI 에이전트가 자동 처리**
- 결과물은 바로 업로드 가능한 영상 파일

---

## 3. 핵심 컨셉

### 🎬 Scene-driven Architecture

STORYCUT의 모든 처리는 **장면(Scene)** 단위를 기준으로 동작합니다.

- 하나의 영상 = 여러 개의 Scene
- 하나의 Scene = 짧은 영상 + 내레이션 + 분위기

이 구조를 통해:
- 영상 생성 안정성 확보
- 음성/영상 길이 제어 용이
- 실패 시 장면 단위 재시도 가능

---

## 4. 사용자 사용 흐름

1. 사용자는 다음 옵션을 입력한다
   - 이야기 장르 (예: 감동, 스릴러, 동화 등)
   - 전체 분위기
   - 영상 스타일 (애니메이션 / 실사풍 / 일러스트)
   - 음악 옵션
   - 전체 영상 길이

2. STORYCUT 에이전트가 자동으로 다음을 수행한다
   - 장면 단위 스토리 생성
   - 장면별 영상 생성
   - 내레이션 음성 생성
   - 배경 음악 선택 또는 생성
   - 영상 합성

3. 최종 결과물
   - youtube_ready.mp4 파일 생성
   - 사용자는 해당 영상을 자신의 유튜브 채널에 업로드

---

## 5. 시스템 아키텍처 개요

```
User Input
  ↓
Story Generator (Scene JSON)
  ↓
Scene Orchestrator
  ↓
Video Generator / TTS / BGM
  ↓
FFmpeg Composer
  ↓
youtube_ready.mp4
```

---

## 6. Scene JSON 계약 (Core Data Model)

STORYCUT의 모든 에이전트는 아래 JSON 구조를 기준으로 동작합니다.

```json
{
  "title": "이야기 제목",
  "genre": "감동",
  "total_duration_sec": 60,
  "scenes": [
    {
      "scene_id": 1,
      "narration": "비가 내리던 밤, 작은 마을은 조용했다.",
      "visual_description": "비 오는 밤의 작은 마을과 가로등",
      "style": "cinematic animation",
      "mood": "차분하고 쓸쓸함",
      "duration_sec": 5
    }
  ]
}
```

## 7. 프로젝트 범위 (MVP 기준)

포함:
- 스토리 생성
- 장면 기반 영상 생성
- TTS 내레이션
- 배경 음악 삽입
- FFmpeg 기반 영상 합성

제외:
- 고급 편집 UI
- 자동 채널 운영
- 조회수 보장 로직

---

## 8. 프로젝트 철학

- STORYCUT은 **창작을 대체하지 않는다**
- STORYCUT은 **창작을 가속한다**
- AI는 도구이며, 크리에이터가 주인이다

---

## 9. Code Agent 작업 지침 (중요)

이 프로젝트의 구현을 담당하는 코드 에이전트는 다음 원칙을 반드시 따른다.

- 모든 로직은 Scene 단위로 처리한다
- 하나의 거대한 AI 호출은 사용하지 않는다
- 각 단계는 독립적이고 재시도 가능해야 한다
- 상용 SaaS 수준이 아닌 MVP 완성을 목표로 한다
